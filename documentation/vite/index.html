Vite

Next-generation frontend tooling. It's fast.

Overview

Vite consists of two main components:

  • Dev server — serves files over native ES modules with instant HMR (Hot Module Replacement)
  • Build command — bundles code via Rolldown for optimized production assets

Browser requirements: Node.js 20.19+ or 22.12+. Modern browsers supporting ES modules for dev; Baseline Widely Available for production.

Installation

# Create a new project (interactive)
npm create vite@latest

# With template flags
npm create vite@latest my-app -- --template vanilla
npm create vite@latest my-app -- --template react
npm create vite@latest my-app -- --template vue

# Add to existing project
npm install -D vite

Available templates: vanilla, vanilla-ts, vue, vue-ts, react, react-ts, react-swc, react-swc-ts, preact, preact-ts, lit, lit-ts, svelte, svelte-ts, solid, solid-ts, qwik, qwik-ts

CLI Commands

CommandDescription
vite / vite devStart dev server (default port 5173)
vite buildBuild for production (outputs to dist/)
vite previewPreview production build locally
vite --port 3000Start dev on port 3000
vite --hostExpose to network (LAN access)
vite --mode stagingOverride env mode
vite build --watchWatch mode for build

Config File

Vite looks for vite.config.js / vite.config.ts in the project root.

// vite.config.js
import { defineConfig } from 'vite'

export default defineConfig({
  // options here
})
// TypeScript
import { defineConfig } from 'vite'
import type { UserConfig } from 'vite'

export default defineConfig({} satisfies UserConfig)
// With intellisense via JSDoc (JS projects)
/** @type {import('vite').UserConfig} */
export default {
  // ...
}

Environment Modes

// Conditional config based on command/mode
export default defineConfig(({ command, mode }) => {
  if (command === 'serve') {
    return {
      // dev-specific config
    }
  } else {
    // command === 'build'
    return {
      // build-specific config
    }
  }
})

// Async config
export default defineConfig(async ({ command, mode }) => {
  const data = await asyncFunction()
  return {
    // ...
  }
})

Server Options

export default defineConfig({
  server: {
    host: true,          // expose to network
    port: 3000,
    strictPort: true,    // fail if port taken
    open: true,          // open browser on start
    https: true,
    cors: true,

    // Proxy API calls
    proxy: {
      '/api': {
        target: 'http://localhost:8000',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api/, ''),
      },
      // Regex proxy
      '^/fallback/.*': {
        target: 'http://jsonplaceholder.typicode.com',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/fallback/, ''),
      },
    },

    // File watching options
    watch: {
      usePolling: true,  // needed in some Docker/WSL envs
    },

    hmr: {
      port: 24678,       // custom HMR port
    },
  },
})

Build Options

export default defineConfig({
  build: {
    outDir: 'dist',         // output directory
    assetsDir: 'assets',    // nested dir for assets
    assetsInlineLimit: 4096, // 4kb — smaller files become base64
    sourcemap: false,        // true | 'inline' | 'hidden'
    minify: 'esbuild',      // 'terser' | 'esbuild' | false
    target: 'modules',      // browser target(s)
    cssMinify: true,
    emptyOutDir: true,      // clear outDir on build
    chunkSizeWarningLimit: 500, // kB

    rollupOptions: {
      input: {
        main: 'index.html',
        nested: 'nested/index.html',
      },
      output: {
        manualChunks: {
          vendor: ['react', 'react-dom'],
        },
      },
    },

    lib: {                  // library mode
      entry: 'src/index.ts',
      name: 'MyLib',
      formats: ['es', 'cjs', 'umd'],
    },
  },
})

Environment Variables

Create .env files in the project root:

# .env             — loaded always
# .env.local        — loaded always, gitignored
# .env.[mode]       — loaded for specific mode
# .env.[mode].local — loaded for specific mode, gitignored

# Variables must be prefixed with VITE_ to be exposed to client code
VITE_API_URL=http://localhost:8000
VITE_APP_TITLE=My App

# Non-prefixed variables are server-only (not exposed)
DATABASE_URL=postgres://...

Access in client code:

console.log(import.meta.env.VITE_API_URL)
console.log(import.meta.env.MODE)      // 'development' | 'production'
console.log(import.meta.env.DEV)       // boolean
console.log(import.meta.env.PROD)      // boolean
console.log(import.meta.env.SSR)       // boolean

TypeScript IntelliSense — create src/vite-env.d.ts:

/// <reference types="vite/client" />

interface ImportMetaEnv {
  readonly VITE_API_URL: string
  readonly VITE_APP_TITLE: string
}

interface ImportMeta {
  readonly env: ImportMetaEnv
}

Load env in config file:

import { defineConfig, loadEnv } from 'vite'

export default defineConfig(({ mode }) => {
  const env = loadEnv(mode, process.cwd(), '')
  return {
    server: {
      port: Number(env.PORT) || 5173,
    },
    define: {
      __APP_VERSION__: JSON.stringify(env.npm_package_version),
    },
  }
})

Static Assets

// Import asset in JS/TS — returns resolved URL string
import logoUrl from './assets/logo.png'
// logoUrl = '/assets/logo.2d8efhg.png' in production

// Import as string (raw text)
import shaderSource from './shader.glsl?raw'

// Import as URL
import workerUrl from './worker.js?url'

// Import as Worker
import MyWorker from './worker.js?worker'
const worker = new MyWorker()

Files in the public/ folder are served at root without hashing — reference as /logo.png, not as imports.

Hot Module Replacement (HMR)

// Accept updates for this module
if (import.meta.hot) {
  import.meta.hot.accept((newModule) => {
    // react to the updated module
  })

  // Accept updates from a dependency
  import.meta.hot.accept('./dep.js', (newDep) => {
    // ...
  })

  // Dispose side effects before module is replaced
  import.meta.hot.dispose((data) => {
    // cleanup (e.g. clear intervals)
  })

  // Mark as not HMR-able; full reload instead
  import.meta.hot.decline()
}

// Invalidate this module and trigger full reload
import.meta.hot.invalidate()

Glob Import

// Import all matching files as a map
const modules = import.meta.glob('./dir/*.js')
// { './dir/foo.js': () => import('./dir/foo.js'), ... }

// Eager loading (included in bundle immediately)
const modules = import.meta.glob('./dir/*.js', { eager: true })

// Named exports only
const modules = import.meta.glob('./dir/*.js', { import: 'setup' })

// As raw strings
const modules = import.meta.glob('./dir/*.txt', { query: '?raw', import: 'default' })

// Dynamic usage
for (const path in modules) {
  modules[path]().then((mod) => {
    console.log(path, mod)
  })
}

SSR Build

// vite.config.js — separate SSR entry
export default defineConfig({
  build: {
    ssr: true,
    rollupOptions: {
      input: 'src/entry-server.js',
    },
  },
})

// package.json scripts
{
  "scripts": {
    "build": "vite build && vite build --ssr src/entry-server.js"
  }
}

Laravel Vite Plugin

// vite.config.js for Laravel
import { defineConfig } from 'vite'
import laravel from 'laravel-vite-plugin'

export default defineConfig({
  plugins: [
    laravel({
      input: ['resources/css/app.css', 'resources/js/app.js'],
      refresh: true,
    }),
  ],
})
<!-- resources/views/app.blade.php -->
@vite(['resources/css/app.css', 'resources/js/app.js'])
// resources/js/app.js
import './bootstrap'
import Alpine from 'alpinejs'
import 'flowbite'

window.Alpine = Alpine
Alpine.start()

})
