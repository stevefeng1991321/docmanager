Alpine.js

A rugged, minimal framework for composing behaviour directly in your markup.

Installation

Via CDN (simplest)

<script defer src="https://cdn.jsdelivr.net/npm/alpinejs@3.x.x/dist/cdn.min.js"></script>

Via npm

npm install alpinejs
import Alpine from 'alpinejs'
window.Alpine = Alpine
Alpine.start()

Quick Start Examples

Counter

<div x-data="{ count: 0 }">
    <button @click="count++">Increment</button>
    <span x-text="count"></span>
</div>

Dropdown

<div x-data="{ open: false }">
    <button @click="open = !open">Toggle</button>
    <div x-show="open" @click.outside="open = false">
        Dropdown content...
    </div>
</div>

Search Filter

<div x-data="{
    query: '',
    items: ['Apple', 'Banana', 'Cherry', 'Date'],
    get filtered() {
        return this.items.filter(i => i.toLowerCase().includes(this.query.toLowerCase()))
    }
}">
    <input x-model="query" placeholder="Search...">
    <ul>
        <template x-for="item in filtered" :key="item">
            <li x-text="item"></li>
        </template>
    </ul>
</div>

x-data

Everything in Alpine starts with x-data. It declares a reactive data object for that HTML region and all its children.

<!-- Object -->
<div x-data="{ open: false, count: 0 }">...</div>

<!-- With methods -->
<div x-data="{ open: false, toggle() { this.open = !this.open } }">...</div>

<!-- With getter (computed) -->
<div x-data="{
    items: [1,2,3],
    get total() { return this.items.reduce((a,b) => a+b, 0) }
}">...</div>

<!-- Empty component (no data needed) -->
<div x-data>...</div>

<!-- Named component (see Alpine.data) -->
<div x-data="myComponent">...</div>

Child components can access parent data. Child declarations with the same property name shadow the parent.

x-show

Toggles display: none based on a boolean expression. The element stays in the DOM.

<div x-show="open">Content</div>

<!-- With transition -->
<div x-show="open" x-transition>Content</div>

<!-- Override !important display rules -->
<div x-show.important="open">Content</div>
Use x-cloak alongside to prevent flash-of-content before Alpine loads.

x-if

Conditionally adds/removes an element from the DOM entirely. Must be used on a <template> tag.

<template x-if="loggedIn">
    <div>Welcome back!</div>
</template>

Unlike x-show, the element is destroyed and recreated — lifecycle hooks run each time.

x-for

Loop over an array. Must be used on a <template> tag. Always provide a :key.

<template x-for="item in items" :key="item.id">
    <li x-text="item.name"></li>
</template>

<!-- With index -->
<template x-for="(item, index) in items" :key="index">
    <li><span x-text="index + 1"></span>. <span x-text="item"></span></li>
</template>

<!-- Range -->
<template x-for="i in 5">
    <span x-text="i"></span>
</template>

x-model

Two-way data binding between an input and a reactive property.

<input x-model="username" type="text">
<textarea x-model="message"></textarea>
<input x-model="accepted" type="checkbox">
<select x-model="country">...</select>

<!-- Modifiers -->
<input x-model.lazy="search">   <!-- sync on change, not input -->
<input x-model.number="age">    <!-- cast to Number -->
<input x-model.trim="name">     <!-- trim whitespace -->
<input x-model.debounce="q">    <!-- debounce sync -->

x-text

Sets the text content of an element to the result of a JavaScript expression.

<span x-text="name"></span>
<span x-text="'Hello ' + name"></span>
<span x-text="price.toFixed(2)"></span>

x-html

Sets the inner HTML of an element. Only use with trusted content — XSS risk with user input.

<div x-html="rawHtml"></div>

x-bind :

Dynamically bind HTML attributes or CSS classes to JavaScript expressions.

<!-- Attribute binding -->
<img :src="imageUrl" :alt="imageAlt">
<input :disabled="isLoading">
<a :href="'/users/' + userId">Profile</a>

<!-- Class binding -->
<div :class="{ 'bg-red-500': hasError, 'text-white': hasError }"></div>
<div :class="isActive ? 'font-bold' : 'font-normal'"></div>
<!-- Merge with static classes -->
<div class="p-4" :class="{ 'bg-red-500': hasError }"></div>

<!-- Style binding -->
<div :style="{ color: textColor, fontSize: size + 'px' }"></div>

x-on @

Listen for DOM events and execute expressions when they fire.

<button @click="count++">Click</button>
<input @input="search = $event.target.value">
<form @submit.prevent="submitForm">

<!-- Keyboard -->
<input @keyup.enter="submit">
<input @keyup.shift.enter="newLine">

<!-- Custom events -->
<div @foo="handleFoo">
    <button @click="$dispatch('foo', { data: 123 })">Fire</button>
</div>

Event Modifiers

ModifierEffect
.preventcalls preventDefault()
.stopcalls stopPropagation()
.outsidefires only for clicks outside the element
.windowlistens on the window object
.documentlistens on the document
.oncehandler fires only once
.debounce[.Nms]debounce (default 250ms)
.throttle[.Nms]throttle (default 250ms)
.selfonly fires for events from the element itself
.passiveimproves scroll performance
.capturefires in the capturing phase
<input @input.debounce.500ms="fetchResults">
<div @scroll.window.throttle.750ms="handleScroll"></div>
<div @click.outside="close"></div>

x-transition

Apply CSS transitions when elements are shown/hidden with x-show or x-if.

<!-- Default transition -->
<div x-show="open" x-transition>...</div>

<!-- Custom classes -->
<div x-show="open"
    x-transition:enter="transition ease-out duration-300"
    x-transition:enter-start="opacity-0 scale-90"
    x-transition:enter-end="opacity-100 scale-100"
    x-transition:leave="transition ease-in duration-200"
    x-transition:leave-start="opacity-100 scale-100"
    x-transition:leave-end="opacity-0 scale-90"
>...</div>

<!-- Duration / delay modifiers -->
<div x-show="open" x-transition.duration.500ms>...</div>

x-init

Run code when Alpine initializes the element. Use $nextTick if you need the DOM to be ready.

<div x-data x-init="console.log('initialized')"></div>

<!-- Async initialization -->
<div x-data="{ users: [] }" x-init="users = await fetchUsers()"></div>

x-cloak

Hides elements until Alpine has finished initializing — prevents flash of unprocessed markup.

<!-- In your CSS: -->
[x-cloak] { display: none !important; }

<!-- In your HTML: -->
<div x-data="{ open: false }" x-cloak>...</div>

x-ref

Mark elements for direct DOM access via $refs.

<input x-ref="searchInput" type="text">
<button @click="$refs.searchInput.focus()">Focus</button>

x-ignore

Prevent Alpine from processing this element and all its children (useful for third-party widgets).

<div x-ignore>
    <!-- Alpine will not process anything here -->
</div>

$el

Reference to the root DOM element of the current component.

<div x-data @click="$el.classList.toggle('active')">...</div>

$refs

Access DOM elements marked with x-ref.

<input x-ref="myInput">
<button @click="$refs.myInput.focus()">Focus Input</button>

$store

Access global stores registered with Alpine.store() from any component.

<!-- Reading from store -->
<div :class="$store.darkMode.on ? 'bg-black' : 'bg-white'">...</div>

<!-- Writing to store -->
<button @click="$store.darkMode.toggle()">Toggle Dark Mode</button>

$watch

Watch a data property and run a callback when it changes.

<div x-data="{ count: 0 }" x-init="$watch('count', val => console.log(val))">
    <button @click="count++">Increment</button>
</div>

$dispatch

Dispatch a custom DOM event from this element (bubbles up the tree).

<button @click="$dispatch('notify', { message: 'Saved!' })">Save</button>

<!-- Parent listens -->
<div @notify.window="showToast($event.detail.message)">...</div>

$nextTick

Execute code after Alpine has finished updating the DOM.

<div x-data="{ open: false }">
    <button @click="open = true; $nextTick(() => $refs.dialog.focus())">Open</button>
    <dialog x-ref="dialog" x-show="open">...</dialog>
</div>

$root

Reference to the nearest ancestor element with x-data.

<div x-data="{ message: 'Hello' }">
    <div>
        <button @click="$root.remove()">Remove component</button>
    </div>
</div>

Alpine.data()

Register reusable component data to avoid repetition across multiple elements.

document.addEventListener('alpine:init', () => {
    Alpine.data('dropdown', () => ({
        open: false,
        toggle() { this.open = !this.open },
        close() { this.open = false },
    }))
})

<!-- Usage -->
<div x-data="dropdown">
    <button @click="toggle">Toggle</button>
    <div x-show="open">Content</div>
</div>

Alpine.store()

Register global reactive state accessible from any component via $store.

document.addEventListener('alpine:init', () => {
    // Object store
    Alpine.store('darkMode', {
        on: false,
        toggle() { this.on = !this.on }
    })

    // Primitive store
    Alpine.store('count', 0)
})

<!-- Access anywhere -->
<button @click="$store.darkMode.toggle()">Toggle Dark Mode</button>
<span x-text="$store.count"></span>
<button @click="$store.count++">Increment</button>
