Tailwind CSS

A utility-first CSS framework for rapidly building custom designs.

Installation with Vite

# 1. Create your project
npm create vite@latest my-project
cd my-project

# 2. Install Tailwind CSS
npm install tailwindcss @tailwindcss/vite

Add the Tailwind plugin to vite.config.ts:

import { defineConfig } from 'vite'
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [tailwindcss()],
})

Add to your main CSS file:

@import "tailwindcss";

Utility-First Fundamentals

Style elements by applying predefined classes directly in your HTML rather than writing custom CSS.

<div class="mx-auto flex max-w-sm items-center gap-x-4 rounded-xl bg-white p-6 shadow-lg">
  <img class="size-12 shrink-0" src="/logo.svg" alt="Logo" />
  <div>
    <div class="text-xl font-medium text-black">ChitChat</div>
    <p class="text-gray-500">You have a new message!</p>
  </div>
</div>

Benefits

  • Faster development — no context-switching between HTML and CSS files
  • Safer changes — modifying a utility only affects that element
  • Non-growing CSS — utilities are reused, CSS doesn't bloat
  • Design constraints — choose from predefined scale instead of magic numbers

Responsive Design

Prefix utilities with breakpoint names to apply styles at specific screen sizes. Mobile-first by default.

PrefixMin-width
sm:640px
md:768px
lg:1024px
xl:1280px
2xl:1536px
<div class="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3">
  <!-- 1 col on mobile, 2 on sm, 3 on lg -->
</div>

Dark Mode

Use the dark: prefix to apply styles when dark mode is active:

<div class="bg-white dark:bg-gray-800 rounded-lg p-6">
  <h3 class="text-gray-900 dark:text-white">Heading</h3>
  <p class="text-gray-500 dark:text-gray-400">Body text</p>
</div>

Hover, Focus & Other States

Prefix utilities with state variants like hover:, focus:, active:, disabled::

<button class="bg-sky-500 hover:bg-sky-700 focus:ring-2 focus:ring-sky-400">
  Save changes
</button>

<input class="border focus:border-blue-500 focus:outline-none focus:ring-1">

<!-- Stack variants for complex conditions -->
<button class="dark:lg:hover:bg-indigo-600">...</button>

Group & Peer Variants

<!-- Style children on parent hover -->
<a href="#" class="group rounded-lg p-8">
  <span class="group-hover:underline">Read more</span>
</a>

<!-- Style siblings on sibling state -->
<input class="peer" type="checkbox">
<label class="peer-checked:font-bold">Option</label>

Arbitrary Values

Use square bracket syntax for one-off values outside your theme:

<!-- Custom color -->
<div class="bg-[#316ff6] text-[14px]"></div>

<!-- Custom grid -->
<div class="grid grid-cols-[24rem_2.5rem_minmax(0,1fr)]"></div>

<!-- With calc() -->
<div class="max-h-[calc(100vh-4rem)]"></div>

<!-- CSS variable -->
<div class="[--gutter:1rem] lg:[--gutter:2rem]"></div>

Class Composition

Multiple utilities targeting the same CSS property compose via CSS variables:

<!-- blur and grayscale both set the filter property via variables -->
<div class="blur-sm grayscale"></div>

<!-- transition with duration and easing -->
<div class="transition duration-300 ease-in-out"></div>

Conflict Management

<!-- Use conditional class, not both -->
<div class={isGrid ? "grid" : "flex"}></div>

<!-- Force a utility with ! -->
<div class="bg-teal-500 bg-red-500!"></div>

Custom CSS with @layer

Extract reusable patterns into custom component classes:

@import "tailwindcss";

@layer components {
  .btn-primary {
    border-radius: calc(infinity * 1px);
    background-color: var(--color-violet-500);
    padding-inline: --spacing(5);
    padding-block: --spacing(2);
    font-weight: var(--font-weight-semibold);
    color: white;

    &:hover {
      @media (hover: hover) {
        background-color: var(--color-violet-700);
      }
    }
  }
}

@layer utilities {
  .content-auto {
    content-visibility: auto;
  }
}
